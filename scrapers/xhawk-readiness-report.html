What does xhawk readiness score mean?

Readiness Score is your Autonomy Gating score. Coding Agents work great if your source code is ready for them. Missing instructions or lack of documentation means your coding agents grep, guess, and ship unreliable code — breaking conventions they never knew existed. This report shows exactly where the gaps are.
Score RangeClassificationRecommended Agent Scope
0 – 49Agent HostileRead-only, suggestions only. Agent can review code and propose changes, but a human should write and apply all edits.
50 – 74Agent CapableSingle-file changes with human review. Agent can edit one file at a time — bug fixes, small features, test additions — but every change needs a human to verify before merging.
75 – 89Agent ReadyMulti-file refactors and feature work. Agent can make coordinated changes across files — add endpoints, update tests, refactor modules — with lighter review.
90 – 100Agent NativeAutonomous tasks end-to-end. Agent can pick up issues, implement features, write tests, and open PRs with minimal supervision. Trust the guardrails you’ve built.

Have you documented what the agent should NOT do?

Why this matters: Agents fail more from doing the wrong thing than from not doing the right thing. Telling an agent "build a REST API" without saying "NEVER do X" means it will confidently do X if it technically works. Prohibitions are your strongest guardrails.

▶ Performance Intent
0 / 15
Have you marked code that looks messy but is intentionally optimized for speed?

Why this matters: Agents love to "clean up" code. If you have a hand-optimized function that looks ugly but is fast on purpose, an agent will happily refactor it into a clean, readable version that runs much slower. Without markers, agents can't tell intentional optimization from tech debt.
What "present" looks like
Your docs say:
"The search indexer uses raw SQL intentionally — do not refactor to the ORM. This is our hot path at P99 < 50ms."
"The cache layer is denormalized by design. Do not 'normalize' it."
What "missing" looks like
You ask the agent to "refactor the data layer." It replaces a hand-tuned SQL query with three chained ORM calls because they're "cleaner." The endpoint goes from 50ms to 800ms. Your users notice.

No performance intent documented. Agents will refactor "ugly but fast" code into "clean but slow" code because they cannot distinguish intentional optimization from tech debt.
